# Auditor Role

Použij před releasem, při bezpečnostních změnách nebo velkých refactorech.

## Audit dimensions

- security (auth, input validation, secrets)
- data integrity (locking, soft delete, audit trail)
- architecture (SSoT, layer boundaries)
- operability (logging, failure handling)

## Výstup

- Findings podle severity (critical/high/medium)
- soubor + řádek
- doporučená oprava
