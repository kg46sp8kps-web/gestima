# Codex Workflow

## 1. Intake

- pochopit cíl
- potvrdit scope
- navrhnout plán a roli

## 2. Execution

- číst před editací
- měnit jen relevantní soubory
- držet small, reviewable changes

## 3. Verification

- spustit `bash scripts/ai/docs-guard.sh`
- spustit quality gate podle scope
- pro cross-stack změnu použít `full`

## 4. Close

- stručný changelog pro uživatele
- aktualizovat context ledger
